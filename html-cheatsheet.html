01

Document Structure

<!DOCTYPE html>

Always the very first line of every HTML file. It declares to the browser that this document uses HTML5. Without it, browsers enter "quirks mode" and render pages inconsistently across devices.

copy <!DOCTYPE html> <html lang="en"> <head> <!-- metadata and settings --> </head> <body> <!-- visible page content --> </body> </html>
The lang attribute on <html> helps screen readers pronounce content correctly and assists search engines in identifying the page language. Always include it.
<head>   <body>

The <head> contains invisible instructions for the browser — character encoding, linked stylesheets, page title, and meta information. The <body> holds everything the user actually sees on screen.

TagContainsVisible to user?
<head>title, meta, link, script referencesNo
<body>headings, paragraphs, images, links, formsYes
<!-- comment -->

HTML comments are notes written for developers. The browser ignores them completely — they never appear on the page. Use them to explain your code, mark sections, or temporarily disable parts of your HTML.

copy <!-- This is a single-line comment --> <!-- This is a multi-line comment. Useful for explaining a complex section or disabling code temporarily. -->
02

Text Elements

<h1>  to  <h6>

HTML provides six heading levels. <h1> is the main page title — use it only once per page. Headings create a logical outline that screen readers and search engines use to understand your content. Never skip levels for visual reasons; if you want bigger or smaller text, use CSS.

copy <h1>Page Title — use only once</h1> <h2>Major Section</h2> <h3>Sub-section</h3> <h4>Sub-sub-section</h4>
Headings communicate meaning, not appearance. Using <h3> because it "looks the right size" is incorrect — use CSS for sizing instead.
<p>

The paragraph tag wraps blocks of text. It is a block-level element, meaning it always starts on a new line. Browsers automatically add spacing above and below each paragraph.

copy <p>This is the first paragraph.</p> <p>This is the second paragraph — it begins on a new line.</p>
<strong>   <em>   <mark>   <code>   <br>   <hr>

Inline elements format text without breaking to a new line. They are used inside block elements like paragraphs.

copy <strong>Bold — marks important text</strong> <em>Italic — adds emphasis</em> <mark>Highlighted text</mark> <code>Inline code snippet</code> <br> <!-- line break — no closing tag --> <hr> <!-- horizontal rule — topic separator -->
TagSemantic meaningVisual effect
<strong>Important contentBold
<em>EmphasisItalic
<mark>Highlighted / relevantYellow background
<code>Code snippetMonospace font
<br>Line breakNew line (no spacing)
<hr>Thematic breakHorizontal line
03

Lists

<ul>   <ol>   <li>   <dl>   <dt>   <dd>

HTML has three list types. Use <ul> when order does not matter (bullet list), <ol> when order matters (numbered list), and <dl> for term-definition pairs. Every list item goes inside an <li> tag.

copy <!-- Unordered list --> <ul> <li>HTML</li> <li>CSS</li> <li>JavaScript</li> </ul> <!-- Ordered list --> <ol> <li>First step</li> <li>Second step</li> </ol> <!-- Definition list --> <dl> <dt>HTML</dt> <dd>HyperText Markup Language — the structure of web pages</dd> </dl>
04

Links & Media

<a>

The anchor tag creates hyperlinks. The href attribute holds the destination. Use target="_blank" to open in a new tab — and always pair it with rel="noopener noreferrer" for security.

copy <!-- External link --> <a href="https://example.com" target="_blank" rel="noopener noreferrer"> Visit Example </a> <!-- Internal page link --> <a href="about.html">About</a> <!-- Jump to section on same page --> <a href="#contact">Go to Contact</a>
<img>

The image tag embeds an image. It is a void element — no closing tag needed. The alt attribute is required: it describes the image for screen readers and displays when the image cannot load.

copy <img src="photo.jpg" alt="A descriptive text for accessibility" width="400" height="300" loading="lazy" >
AttributePurposeRequired?
srcPath to the image fileYes
altAccessible descriptionYes
width / heightPrevents layout shift during loadRecommended
loading="lazy"Delays load until image is near viewportOptional
05

Tables

<table>   <thead>   <tbody>   <tr>   <th>   <td>

Tables display data in rows and columns. Always separate header rows (<thead>) from data rows (<tbody>) — this improves accessibility and allows browsers to scroll the body independently. Use colspan and rowspan to span cells across multiple columns or rows.

copy <table> <thead> <tr> <th>Name</th> <th>Role</th> </tr> </thead> <tbody> <tr> <td>Fayza</td> <td>Designer</td> </tr> </tbody> </table>
Tables are for tabular data only — not for page layout. Use CSS Grid or Flexbox for layout instead.
06

Forms & Inputs

<form>   <input>   <label>   <select>   <textarea>   <button>

Forms collect user input. Every input should have a matching <label> linked via matching for and id values — this is required for accessibility and makes the label clickable.

copy <form action="/submit" method="POST"> <!-- Text input --> <label for="name">Full Name</label> <input type="text" id="name" name="name" placeholder="Your name" required> <!-- Email input --> <input type="email" placeholder="you@email.com"> <!-- Dropdown --> <select> <option>Option A</option> <option>Option B</option> </select> <!-- Multi-line text --> <textarea rows="4" placeholder="Your message..."></textarea> <button type="submit">Send</button> </form>
input type=What it does
"text"Single-line free text
"email"Validates email format automatically
"password"Masks typed characters
"number"Numeric input with increment controls
"checkbox"Toggle true / false
"radio"Select one option from a group
"file"File upload selector
"date"Date picker calendar UI
07

Semantic HTML

<header>   <nav>   <main>   <section>   <article>   <aside>   <footer>

Semantic elements carry meaning — they tell the browser, screen reader, and search engine what the content represents, not just where it sits. Always prefer semantic tags over generic <div> containers.

copy <header> <nav> ... navigation links ... </nav> </header> <main> <section> <article> ... self-contained content ... </article> </section> <aside> ... sidebar content ... </aside> </main> <footer> ... copyright, links ... </footer>
TagUse for
<header>Page or section header, logo, top nav
<nav>Navigation menus
<main>Primary content of the page — use only once
<section>A thematic group of content with a heading
<article>Self-contained piece (blog post, card, comment)
<aside>Sidebar or supplementary content
<footer>Page footer — copyright, contact, links
<figure> / <figcaption>Image or diagram with an optional caption
Use <div> only when no semantic element fits. Before reaching for it, ask: is there a more descriptive tag for this content?
08

Head & Meta Elements

<meta>   <title>   <link>   <script>

Elements inside <head> configure the page for browsers, search engines, and social media. These are invisible to users but critical for responsiveness, performance, and discoverability.

copy <head> <!-- Character encoding — always first --> <meta charset="UTF-8"> <!-- Required for responsive design on mobile --> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- Browser tab title --> <title>My Website</title> <!-- SEO page description --> <meta name="description" content="My personal portfolio"> <!-- Link to external CSS --> <link rel="stylesheet" href="style.css"> <!-- Favicon --> <link rel="icon" href="favicon.ico"> </head> <!-- Script at end of body for performance --> <script src="script.js"></script>
The viewport meta tag is what makes your page responsive on mobile. Without it, mobile browsers render the page as a shrunken desktop version — users must zoom in to read.
